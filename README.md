# 期货程序化交易系统功能标准符合性评估

## 考核目标

考察候选人借助 AI 工具完成实际工程任务的能力。

本仓库已提供基于 openctp (TTS) 仿真环境的连通性测试代码（C++ / Python），覆盖了评估表 21 项中的第 1 项。你需要在 **30 分钟** 内：

1. 选择 C++ 或 Python 其中一种语言
2. 补充实现评估表中剩余的功能项目，并编写自动化测试
3. 回填评估表的“自评结果”列

评估表位于 `docs/期货程序化交易系统功能标准符合性评估表.xlsx`，详见“软件自评估表”Sheet。

## 考核任务

### 核心要求

选择 **Python(`py/`) 或 C++(`cpp/`)** 其中一种语言，补充代码实现上述 **"待实现"** 的所有功能点，并为每个功能点编写对应的自动化测试。不要求两种语言都实现。

### 具体任务

#### 1. 基础交易功能 (序号 2-4)
- 实现开仓、平仓、撤单操作
- 基于现有 `test_td.py`（或 `test_td.cpp`）扩展，使用 TTS 7x24 环境
- 测试需验证报单回报、成交回报、撤单回报是否正确

#### 2. 异常监测 (序号 5-10)
- 连接状态监测：检测启动、正常运行、断开、重连状态
- 报撤单笔数统计：记录账号报单/撤单笔数
- 重复报单检测：识别相同合约、方向、价格、数量的重复指令

#### 3. 错误防范 (序号 11-16)
- 交易指令前置检查：合约代码、最小变动价位、最大委托手数
- 错误提示展示：资金不足、持仓不足、市场状态不可用

#### 4. 应急处置 (序号 17-19)
- 暂停交易功能（停止下单）
- 批量撤单：部分撤单 / 全部撤单

#### 5. 日志记录 (序号 20-21)
- 结构化日志记录：交易日志、系统运行记录、监测记录、错误提示记录

### 技术约束

- **交易环境**: openctp TTS 7x24 仿真环境（`.env` 已配置）
- **Python 依赖**: `openctp-ctp` (已安装)
- **C++ 依赖**: `cpp/deps/` 下的 CTP 头文件和库
- **测试框架**: Python 用 pytest，C++ 通过 ctest 运行
- **合约选择**: 可使用 `ag2606`, `IF2604` 等

### 交付物

1. 所有新增代码通过 `ctest` 或 `pytest` 自动化测试
2. 评估表 "严重" 等级项目必须全部实现
3. "建议" 等级项目尽量实现
4. 代码风格与现有代码保持一致
5. **回填评估表**：30 分钟结束时，打开 `docs/期货程序化交易系统功能标准符合性评估表.xlsx` 的 "软件自评估表" Sheet，将 **"自评结果"** 列逐项填写完成情况：
   - `符合` — 功能已实现且测试通过
   - `不符合` — 功能未实现或测试未通过
   - `不适用` — 软件不支持该功能（仅限"建议"等级项目）

   评估表即为最终交付物，未回填视为未完成。

## 环境准备

```bash
# === 选择 Python ===
cd py && uv sync && uv pip install pytest pytest-timeout
cd .. && mkdir -p build && cd build
cmake .. -DLANG=py && ctest --output-on-failure

# === 选择 C++ ===
cd cpp && bash setup_deps.sh      # 一键下载 SDK
cd .. && mkdir -p build && cd build
cmake .. -DLANG=cpp && make && ctest --output-on-failure

# === 两者都跑 ===
mkdir -p build && cd build
cmake .. -DLANG=all && make && ctest --output-on-failure

# 也可以单独用 pytest
cd py && pytest -v
```
